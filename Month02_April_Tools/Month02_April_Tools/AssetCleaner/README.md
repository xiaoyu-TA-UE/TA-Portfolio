# Maya 资产清理工具 (Asset Cleaner)

**版本**: 2.0 Final | **作者**: [小宇] | **日期**: 2026年4月

## 📖 简介

批量执行安全的资产标准化清理操作。与 `asset_perf_check.py` 配合使用，形成 **“检查-修复”闭环**。

**设计原则**：只做清理，不做检查。只处理安全、可逆、可批量的操作。

## 🧩 清理步骤（不可更改顺序）

| 顺序 | 步骤         | 理由                                                   |
| :--- | :----------- | :----------------------------------------------------- |
| 1    | **冻结变换** | 将位移/旋转/缩放归零或归一，防止导入引擎后位置错位     |
| 2    | **删除历史** | 清除构造历史（包括冻结变换产生的新节点），减小文件体积 |
| 3    | **归轴心点** | 将轴心移到模型底部中心，便于引擎中放置和对齐           |
| 4    | **锁定属性** | 防止清理后被误操作                                     |

**为什么顺序不可改？** 冻结变换会产生新的历史节点，所以必须在删除历史之前执行；归轴心需要拿到模型真实包围盒，所以必须在变换冻结之后执行。

## 🚀 使用方法

1. 在 Maya 中**手动选中**需要清理的模型。
2. 在脚本编辑器中打开 `clean_assets.py`，点击执行。
3. 观察脚本编辑器和 Maya 视口中的清理反馈。

> ⚠️ 本工具**不会**自动处理 UV、非流形、面数问题。这些问题需要美术根据 `asset_perf_check.py` 的报告手动修复。

## 📂 文件结构

```
clean_assets.py
├── CONFIG                    # 配置中心（各步骤开关）
├── freeze_transform()        # 冻结变换
├── delete_history()          # 删除构造历史
├── center_pivot()            # 归轴心点至模型底部中心
├── lock_transform_attributes() # 锁定变换属性
├── clean_selected_assets()   # 主清理流程
└── __name__ == "__main__"   # 脚本入口
```

## 🛠️ 与工具链的关系

本工具是 **“修复”** 端。配合 `asset_perf_check.py`（诊断端），形成完整闭环：

1. `asset_perf_check.py` 扫描资产 → 生成报告。
2. 根据报告手动选中问题资产。
3. 运行本工具批量修复变换冻结、历史清理等可自动化的问题。

## 📜 代码规范遵循

严格遵循《代码风格生成规范》：

- 变量名见名知意
- 每个物体每步操作独立 `try-except`，单个报错不影响整体
- 所有开关集中管理在 `CONFIG` 字典中
