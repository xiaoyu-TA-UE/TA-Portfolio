# Asset Performance Checker (资产性能检查工具)

**版本**: 9.0 Final | **作者**: [小宇] | **日期**: 2026年4月

## 📖 简介

在 DCC 环节（Maya）提前发现性能问题的自动化诊断工具。对选中的模型执行多维度检查，生成深色主题的 HTML 可视化报告。

**核心理念**：把性能规范从“靠人记”变成“靠工具自动执行”。

## 🧩 功能模块

| 模块           | 说明                                                         |
| :------------- | :----------------------------------------------------------- |
| **资产获取**   | 支持手动选中模型或一键全场景扫描，未选中时弹窗引导用户选择   |
| **变换检查**   | 检查位移/缩放是否已冻结（使用分步显式判断，禁用 `any()`/`all()`） |
| **UV检查**     | 检查 UV 是否在 0-1 象限内（防止 Lightmap 烘焙失败）          |
| **非流形检查** | 检查是否存在坏边、坏点（防止 LOD 生成失败和法线错误）        |
| **面数统计**   | 统计每个模型的面数和顶点数，超过阈值自动标记                 |
| **空间分析**   | 计算资产间距，防止重叠导致的 Z-fighting 和 Overdraw          |
| **HTML 报告**  | 生成深色主题专业报告，包含扫描概览、资产详情和间距分析表格   |

## 🚀 使用方法

### 在 Maya 脚本编辑器中运行

1. 选中需要检查的模型（可选）。
2. 在脚本编辑器中打开 `asset_perf_check.py`，点击执行。
3. 如果未选中任何模型，脚本会弹窗询问扫描模式。
4. 选择报告保存路径后，自动生成并打开 HTML 报告。

### 命令行模式

```bash
mayapy asset_perf_check.py --check    # 仅检查并生成报告
mayapy asset_perf_check.py --clean    # 检查并自动清理问题资产
```

## 📂 文件结构

```
asset_perf_check.py
├── CONFIG              # 配置中心（所有阈值、路径、开关）
├── get_target_assets() # 资产获取（支持手动选择和全场景扫描）
├── check_transform_frozen()  # 变换冻结检查
├── check_uv_range()          # UV 越界检查
├── check_non_manifold()      # 非流形几何检查
├── get_mesh_stats()          # 面数/顶点数统计
├── evaluate_asset_status()   # 综合状态评估
├── check_asset_spacing()     # 3D 空间分析
├── generate_html_report()    # HTML 报告生成
├── main()                    # 主控逻辑
└── __name__ == "__main__"   # 脚本入口
```

## 🛠️ 与工具链的关系

本工具是 **“诊断”** 端。配合 `clean_assets.py`（修复端），形成完整的 **“检查-修复”闭环**：

1. 运行本工具扫描资产 → 生成报告定位问题。
2. 根据报告手动选中问题资产。
3. 运行 `clean_assets.py` 批量修复可安全自动化的问题。

## 📜 代码规范遵循

严格遵循《代码风格生成规范》：

- 变量名见名知意（`translate_x` 而非 `tx`）
- 使用显式 `for` 循环，禁用 `any()`/`all()`/嵌套列表推导式
- 所有函数包含 `try-except` 异常处理
- 所有阈值配置集中管理，绝不硬编码
- 关键步骤用中文注释解释性能影响
